（1）Fork：将原作者的项目复制到自己的仓库里面。Fork完成的项目完全属于自己，可进行增加，删除，更改，重命名，但这都不会破坏原作者            的项目。
     Pull Request：其他人对原作者项目修改后，想要将其贡献出去就要使用PullRequest。一旦提交后其他人就可以Revive这个更改后的项目。
                   修改后的项目被有权限的人审核通过后，就可以被整合至原项目里。
